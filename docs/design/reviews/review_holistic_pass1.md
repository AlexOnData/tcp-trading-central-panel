## Summary

The package is substantial, internally well-organised, and largely aligned across the three design documents and four diagrams. However, several genuine cross-document inconsistencies must be resolved before Etapa 1 can be closed: a role-name contradiction between `ai_sequence.mmd` and the only DB roles that actually exist (`tcp_ai_assistant`, `tcp_generator`, `tcp_admin`); two conflicting BACPAC schedules (Function App timer Sunday 08:00 RO vs. GitHub Actions Sunday 02:00 UTC); a permissions contradiction between NFR-SEC-05 / ┬º7.3 (which both promise `db_datareader` only) and the actual grant model (which adds `INSERT, UPDATE` to the same MI through `tcp_generator`); the KPI catalogue references field names (`pnl_eur`, `capital_baseline_eur`) and dimensions (`dim_Markets.instrument_name`, `dim_TradingFloors.floor_name`, `dim_OrderType.order_type_name`, `dim_Sessions.session_name`) that do not exist in the schema; and a couple of small but visible English-only violations in committed diagrams/seed data. None of these is structural ÔÇö they are surface-level cleanups that should take an hour to fix.
